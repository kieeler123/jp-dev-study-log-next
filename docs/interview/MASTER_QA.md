A.
フロントだけでなく API設計・DB連携・デプロイ・型エラー対応まで含めて、実務に近い一連の流れを見せられるからです。

🇯🇵 Q7. MongoDB接続で苦労した点は？

A.
MongoDB Atlas の接続文字列（srv）の設定と環境変数管理です。
querySrv EBADNAME _mongodb._tcp... のようなエラーが出た時は、URI・cluster名・DNS設定を見直しました。

🇰🇷

MongoDB Atlas 연결 문자열(srv) 설정과 환경변수 관리였습니다. querySrv EBADNAME 에러가 나왔을 때 URI/클러스터명/DNS 설정을 점검했습니다.

🇯🇵 Q8. Vercelのビルドエラー（params Promise）をどう解決しましたか？

A.
Next.jsのRoute Handlerでは params の型が環境によって Promise になるケースがあり、
シグネチャを params: Promise<{ id: string }> に合わせて await params で取り出すように修正しました。
ローカルで npm run build も通して確認しました。

🇰🇷

Next.js Route Handler에서 params 타입이 Promise로 기대되는 경우가 있어, params: Promise<{id:string}>로 맞추고 await params로 꺼내도록 수정했습니다. 로컬 build 통과도 확인했습니다.

🇯🇵 Q10. Trashはなぜ isTrashed方式？

A.
誤削除対策として復元を可能にするためです。
削除＝状態変更、完全削除＝別操作に分けることで安全性と責務分離を確保しました。

🇰🇷

실수 삭제 대비 복구를 위해서입니다. 삭제=상태 변경, 완전삭제=별도 조작으로 분리해 안전성과 책임 분리를 확보했습니다.

🇯🇵 Q12. イベント委譲を使った理由は？

A.
ボタンが増減してもイベント管理が簡単で、パフォーマンス的にも有利だからです。
closest() を使ってクリック対象を特定しています。

🇰🇷

버튼이 동적으로 늘어나도 이벤트 관리가 쉽고 성능상도 유리해서입니다. closest()로 클릭 대상을 특정했습니다.
